제 2장. 연산자(Operator)와 비트 핵심 정리

🎯 오늘의 학습 목표

1. 연산자와 피연산자의 개념을 이해하고 구별할 수 있습니다.
2. 자바에서 제공하는 다양한 연산자의 종류와 쓰임새를 파악합니다.
3. 비트 시프트(Shift) 연산의 원리를 이해하고 프로그래밍에서의 활용 이유를 설명할 수 있습니다.

1. 연산자란 무엇인가요?

프로그래밍에서 데이터를 계산하거나 가공하기 위해 사용하는 모든 기호를 연산자(Operator)라고 하며, 그 연산의 대상이 되는 데이터를 피연산자(Operand)라고 부릅니다.

💡 쉽게 이해하는 요리 비유

주방에서 요리를 할 때를 생각해 봅시다. '도마 위의 식재료'가 있고, 이를 다듬는 '칼이나 믹서기 같은 요리 도구'가 있습니다.

• 피연산자(Operand) = 요리의 대상이 되는 식재료 (예: 숫자, 문자 등)
• 연산자(Operator) = 식재료를 가공하는 요리 도구 (예: +, -, *, / 등)

예를 들어 1 + 3 이라는 연산식이 있다면 다음과 같이 구분됩니다.

  • 연산자: + (더하기 동작을 수행하는 기호)
  • 피연산자: 1과 3 (더해지는 실제 데이터)

2. 자바 연산자의 종류

자바 언어에서는 목적에 따라 다양한 연산자(요리 도구)를 제공합니다. 아래의 표를 통해 한눈에 정리해 봅시다.

연산자 종류 대표 기호 기능 및 설명
1. 산술 연산자 +, -, *, /, % 더하기, 빼기, 곱하기, 나누기 및 나머지를 구하는 가장 기본적인 수학 계산
2. 관계(비교) 연산자 >, <, ==, != 두 데이터의 크기를 비교하여 참(true) 또는 거짓(false)을 판단
3. 조건(삼항) 연산자 ? : 조건식이 참일 때와 거짓일 때 각각 다른 결과값을 선택하게 함
4. 논리 연산자 &&, ||, ! 그리고(AND), 또는(OR), 반대(NOT) 등 여러 조건을 조합하여 판단
5. 대입(복합) 연산자 =, +=, -= 오른쪽의 계산 결과를 왼쪽 변수에 대입하거나 누적하여 저장
6. 증감 연산자 ++, -- 변수에 저장된 값을 딱 1만큼 증가시키거나 감소시킴
7. 비트 논리 연산자 &, |, ^, ~ 컴퓨터 메모리에 저장된 2진수(0과 1) 상태에서 직접 논리 계산 수행

3. 컴퓨터의 언어, 비트(Bit)와 시프트(Shift) 연산자

사람은 0부터 9까지의 숫자를 쓰는 10진수를 사용하지만, 컴퓨터는 전기가 켜지고(1) 꺼지는(0) 두 가지 상태만 아는 2진수의 세계에 살고 있습니다. 이 0과 1이 들어가는 컴퓨터 메모리의 가장 최소 단위를 비트(Bit)라고 합니다.

시프트(Shift) 연산자는 이 2진수 비트들을 통째로 왼쪽이나 오른쪽으로 한 칸씩 밀어버리는 독특하고 강력한 연산자입니다.

3.1 왼쪽 시프트 연산자 (<<) : 2를 곱하는 마법

비트를 통째로 왼쪽으로 이동시킵니다. 밀려나간 맨 왼쪽 비트들은 버려지고, 새로 생겨난 맨 오른쪽 빈자리는 항상 0으로 채워집니다.

💡 핵심 공식: n << m 은 숫자가 왼쪽으로 가면서 n × 2m의 결과가 됩니다. 즉, 한 칸 밀 때마다 숫자가 2배씩 커집니다!

실제 예시 (3 << 2)

  • 10진수 숫자 3은 2진수로 0011입니다.
  • 이를 왼쪽으로 2칸(<< 2) 밀면 1100이 됩니다.
  • 2진수 1100을 다시 우리가 아는 10진수로 바꾸면 12가 됩니다. (3 × 2 × 2 = 12)

3.2 오른쪽 시프트 연산자 (>>) : 2로 나누는 마법

비트를 통째로 오른쪽으로 이동시킵니다. 밀려나간 맨 오른쪽 비트들은 잘려서 버려집니다. 이때 맨 왼쪽 빈자리에는 숫자의 부호(양수면 0, 음수면 1)를 유지하기 위해 원래 값의 가장 왼쪽 비트(부호 비트)와 같은 값이 채워집니다.

💡 핵심 공식: n >> m 은 숫자가 오른쪽으로 가면서 n ÷ 2m의 몫이 됩니다. 즉, 한 칸 밀 때마다 숫자가 반으로 줄어듭니다!

실제 예시 (12 >> 2)

  • 10진수 숫자 12는 2진수로 1100입니다.
  • 이를 오른쪽으로 2칸(>> 2) 밀면 0011이 됩니다.
  • 2진수 0011을 다시 10진수로 바꾸면 3이 됩니다. (12 ÷ 2 ÷ 2 = 3)

3.3 왜 복잡하게 비트를 밀어서 연산하나요?

  1. ⚡ 압도적인 속도 향상: 컴퓨터 하드웨어 구조상 일반 곱셈/나눗셈 알고리즘보다 비트를 옆으로 쓱 미는 시프트 연산의 처리 속도가 훨씬 빠릅니다.
  2. 💾 메모리 절약: 하나의 정수형 변수 안에 여러 개의 참/거짓(Boolean) 상태 데이터를 비트 단위로 빽빽하게 채워 넣어 기억 공간을 극한으로 아낄 수 있습니다. (비트 필드 기법)
  3. 🔐 보안 및 암호화: 데이터를 알아볼 수 없게 자리를 뒤섞고(암호화), 다시 원래대로 되돌리는(복호화) 연산 구조에 매우 유용하게 쓰입니다.
  4. 🧮 거듭제곱 계산: 2*2*2*2를 구하는 것보다 1을 왼쪽으로 4칸 미는 1 << 4 연산이 코드가 간결하고 빠릅니다.

Q1. int a = 5; 변수에 a += 3; 연산을 수행했습니다. 연산 후 a에 저장된 최종 값은 무엇일까요?

정답: 8 (a = a + 3 과 같은 의미이므로 5 + 3 = 8이 됩니다.)

Q2. 비트 연산식 2 << 3의 연산 결과는 10진수로 얼마일까요?

정답: 16 (숫자 2를 왼쪽으로 3칸 밀었으므로 2 × 2 × 2 × 2 = 16이 됩니다.)
