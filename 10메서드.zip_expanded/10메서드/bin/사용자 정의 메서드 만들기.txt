

메서드?
- 특정 기능을 처리하기 위한 명령들을 묶어 놓은 작은 프로그램 단위를 "메서드" 라고 합니다.



사용자 정의 메서드 ?
- 개발자가 필요에 따라 직접 만드는 메서드입니다.
- 필요한 작업을 미리 만들어두고 필요할 때마다 가져다 쓸 수 있습니다.



"사용자 정의 메서드" 만들기 작성 문법

	  public  static  void       main(String[]  args ) {
	  		
	  		System.out.println("안녕");
	  		
	  }
	  
      접근제어자  수정자  반환자료형  메소드명(자료형 변수명, ...) { 
       
          메소드가 해야 할 일;							  <---메서드의 몸체 
          return 결과값;
          
      }	
	  
	  
	  ******** 위 메서드 작성 문법을 나누어서 생각 해보면 아래와 같다.********
	  ====================================================================

    	{  
          메소드가 해야 할 일;							  <---메서드의 몸체

메서드에 데이터를 전달해주는 방법


	그림1
		static void hello_func() {
		
		
		}
	
	그림1 설명
		hello_fnc메서드는 호출할 때 ()괄호 사이에 아무런 데이터를 넘겨주지 않고 
		{   } 사이의 구문을 처리 합니다.
		이러한 메서드를 매개변수가 없는 메서드라고 합니다.
		
		
	그림2
		static void sum(int n) {
		
		
		}
	
	그림2 설명
		매개변수 n은 메서드를 호출할 때 넘어온 값을 담아두기 위한 기억 장소로서 
		이 변수를 매개변수라고 합니다.
		메서드를 정의하는 측에서는 매개변수 n에 어떤 값이 전달될지 알 수 없고,
		전달하는 값의 종류가 int  n 매개변수로 전달 받는 값이 정수이면? 
		int n 매개변수로 정수를 전달 받아 {  }중괄호 내부에서 사용할수 있다. 
	   
					
	그림3
		sum(5);
		sum(10);
			 
	그림3 설명
		하지만, 매개변수가 있는 메서드를 호출해서 메서드의 기능 코드를 사용하려는 쪽에서는 
		반드시 메서드를 호출해서 부를때 특정한 인수 값(5 또는 10)을 넘겨주어야합니다.		 
		메서드를 호출할때 기술한 5나 10이 실제로 메서드 { }중괄호 내부에서 기능을 만들때 사용할 값이 되므로
		5나 10을 인수라고 합니다.
	
	요약 :   인수   -> 메서드 호출시 매개변수로 전달 하는 값
	       매개변수 -> 메서드 외부에서 받은 값을 전달 받아 사용할 변수 
	
	[예제] 주어진 값까지의 자연수의 합을 구하기 - 파일명 : MethodEx02.java
	
	
	
	
	
결과 값을 되돌리는 방법

	그림1
		static void a() {
		
		  몸체 부분의 내용;
		  
		  return 결과값;
		
		}
		
	그림1 설명
		a메서드가 호출되면 메서드 몸체 부분의 내용이 수행된 후에
		결과값을 가지고  a메서드를 호출한 코드 쪽으로 되돌아 가게할 경우 사용하는 것이 return문입니다.
		return문 다음에는 상수나 변수나 수식등을 사용할 수 있습니다.
		이때 주의할 점은 reutn문을 이용하여 되돌릴 수 있는 값은 하나라는 것입니다.
	
	
	그림2
		static int a(){
		
		  몸체 부분의 내용;
		  
		  return 1;
		
		}
	
	그림2 설명
		메서드의 자료형 int와  return문 다음에 오는 데이터1의 자료형이 일치 해야 합니다.
